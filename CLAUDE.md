# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Static lab website (HTML/CSS/JS) for the Cai Lab at Texas A&M. The publications page and main index are **generated by MATLAB scripts** in `src/` — do not hand-edit `publications.html`, `selected_publications.html`, or `index.html` directly.

## Regenerating the Website

All scripts run from the `src/` directory in MATLAB (requires internet access to fetch metadata):

```matlab
cd src
addpath('pmidoi\')

% Step 1 – refresh publications HTML from PubMed / DOI metadata
% (fetches live data; re-fetches highlighted entries every time)
e1d_pmidoi2html_writemerge   % writes ../publications.html and ../selected_publications.html

% Step 2 – rebuild index.html by merging template + news + publications + people
e2_mergefiles                % writes ../index.html
```

Run both steps in sequence after any change to `ids.txt`, `news_source.txt`, `people.html`, or `index_template.html`.

## Publication Pipeline Architecture

1. **`src/ids.txt`** — master list of PMIDs and DOIs, one per line. Append `*` to mark a preprint. Order determines display order.
2. **`src/e0_pmidoilist.m`** — shared setup: reads `ids.txt`, strips `*`, builds `AAv` (all IDs) and `ispreprint` mask, defines `Highlighted` (a hardcoded list of featured IDs).
3. **`src/pmidoi/i_pmid2html.m`** — fetches PubMed record for a PMID, parses AU/TI/SO/DOI fields, returns `<li>` HTML string.
4. **`src/pmidoi/i_doi2html.m`** — fetches `https://doi.org/<DOI>`, extracts `<meta name="citation_*">` tags, returns `<li>` HTML string. Used for DOI-only entries (arXiv, bioRxiv, non-PubMed journals).
5. **`src/paperhtml/`** — cache directory; each entry is stored as `matlab.lang.makeValidName(<id>)`. Highlighted entries and those with a matching `.wav` file are always re-fetched; others are served from cache.
6. **`src/e1d_pmidoi2html_writemerge.m`** — orchestrates the loop, writes `src/publications.html` (debug copy) and `../publications.html` + `../selected_publications.html` (production).
7. **`src/e2_mergefiles.m`** — reads `index_template.html` and splices in `news_aa.html` (`<<<1>>>`), `publications.html` (`<<<2>>>`), and `people.html` (`<<<3>>>`) at placeholder tokens to produce `../index.html`.

## Adding / Updating Content

| Task | What to edit |
|---|---|
| Add a new publication | Append PMID or DOI to `src/ids.txt`; re-run `e1d_pmidoi2html_writemerge` |
| Feature a publication | Add its ID to the `Highlighted` array in `src/e0_pmidoilist.m` |
| Add/edit news item | Edit `src/news_source.txt` (alternating date line / text line); re-run `e2_mergefiles` |
| Edit people section | Edit `src/people.html`; re-run `e2_mergefiles` |
| Change site layout/nav | Edit `src/index_template.html`; re-run `e2_mergefiles` |

## Audio Abstracts

If a `.wav` file exists at `../wav/<makeValidName(id)>.wav`, the pipeline embeds an `<audio>` element in that paper's `<li>`. Drop `.wav` files in the `wav/` directory and re-run `e1d_pmidoi2html_writemerge`.

## Highlighted vs. Normal Entries

- Normal entries: HTML served from `src/paperhtml/` cache; no network call on re-run.
- Highlighted entries: always re-fetched from PubMed/DOI on every run to pick up citation count updates (Dimensions badge via `data-doi`).
- `selected_publications.html` contains only highlighted entries.
